# Continuous Learning — 패턴 발굴 워크플로우

반복적으로 같은 코드 패턴, 설정, 디버깅을 하고 있다면 스킬로 추출할 신호다.
이 워크플로우는 Git 히스토리와 현재 코드베이스를 분석해 스킬화할 패턴을 찾는다.

> 소스: 내부 워크플로우 (외부 라이브러리 의존 없음)
> 검증일: 2026-04-20

---

## 언제 사용하나

- 새 라이브러리·API를 반복 사용하는데 스킬이 없을 때
- 같은 에러·설정을 여러 번 해결했을 때
- 팀에서 공통으로 쓰는 패턴이 문서화되지 않았을 때
- 한 달마다 주기적으로 스킬 목록을 점검할 때

---

## 4단계 워크플로우

### 단계 1: 히스토리 스캔

최근 커밋에서 반복 등장하는 키워드를 추출한다.

```bash
# 최근 30일 커밋 메시지 분석
git log --since="30 days ago" --oneline

# 자주 수정된 파일 TOP 10
git log --since="30 days ago" --name-only --format="" | sort | uniq -c | sort -rn | head -10

# 특정 패턴의 파일 변경 이력
git log --all -S "useQuery" --oneline   # TanStack Query 사용 추가된 커밋
git log --all -S "Arc<dyn" --oneline    # Rust DI 패턴 추가된 커밋
```

### 단계 2: 패턴 분류

스캔 결과를 3가지 유형으로 분류한다.

| 유형 | 신호 | 스킬 가치 |
|------|------|-----------|
| **A. 반복 설정** | 같은 설정 코드가 여러 파일에 복붙 | 높음 |
| **B. 반복 디버깅** | 같은 에러를 여러 번 해결 | 높음 |
| **C. 반복 패턴** | 같은 코드 구조가 여러 곳에 등장 | 중간 |

**스킬 가치 판단 기준:**
- 앞으로도 반복될 가능성이 높은가?
- 공식 문서를 매번 찾아봐야 하는 내용인가?
- 한 번 잘못 쓰면 버그가 나는 주의사항이 있는가?

### 단계 3: 후보 선정

스킬화할 후보를 다음 형식으로 정리한다.

```markdown
## 스킬 후보 목록

| 후보 | 근거 (반복 횟수/파일) | 카테고리 | 우선순위 |
|------|----------------------|----------|----------|
| xxx  | 5회 반복, 3개 파일   | frontend | 높음     |
| yyy  | 에러 3회 동일 해결   | backend  | 높음     |
```

우선순위 기준:
- 높음: 반복 3회 이상, 공식 문서 복잡
- 중간: 반복 2회, 패턴 명확
- 낮음: 반복 1회지만 향후 재사용 예상

### 단계 4: 스킬 생성

우선순위 높음 후보부터 `skill-creator` 에이전트에 위임한다.

```
사용자 → (이 워크플로우로 후보 선정) → skill-creator 에이전트 호출:
"[후보 스킬명] 스킬 만들어줘. 공식 문서 검증 후 SKILL.md 작성해줘."
```

---

## 현재 스킬 목록과 대조

새 후보를 선정하기 전, 기존 스킬과 겹치지 않는지 확인한다.

```bash
ls .claude/skills/frontend/    # 프론트엔드 스킬 목록
ls .claude/skills/backend/     # 백엔드 스킬 목록
ls .claude/skills/architecture/
ls .claude/skills/devops/      # 빌드·배포 스킬 목록
ls .claude/skills/meta/        # 프로젝트 관리 스킬 목록
```

겹치는 스킬이 있다면 신규 생성보다 기존 스킬 업데이트가 맞다.

## 주기적 실행 권장

| 주기 | 대상 |
|------|------|
| 매월 말 | 전체 히스토리 스캔 + 후보 선정 |
| 새 라이브러리 도입 후 | 해당 라이브러리 패턴 스킬화 |
| 같은 에러 2회 이상 | 즉시 디버깅 패턴 스킬화 |
