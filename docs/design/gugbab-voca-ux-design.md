## 1. 앱 컨셉

### 한 줄 정의
**CEFR 6단계(A1~C2) 영어 회화 단어·문장을 SRS(SM-2) 기반 플래시카드로 학습하는 1인용 PWA.**

### 핵심 가치 제안
| 가치 | 의미 |
|---|---|
| **단순함** | 학습 동작은 *뒤집기 → 알았음/모르겠음* 단 2개 액션. 추가 사고 부담 X |
| **자동 복습** | SM-2가 잊을 만한 시점에 카드를 자동 재등장 — 사용자는 진도 관리 불필요 |
| **항상 사용 가능** | PWA·정적 호스팅·IndexedDB로 *오프라인·로그인 없음·동기화 없음* |
| **자투리 시간 친화** | 5장·20장 단위 세션. 출퇴근·점심 자투리 5~15분에 끝남 |

### 타깃 사용자
- **본인 1인용**. 박찬호(직장인). 다중 사용자·계정 개념 없음
- 모바일(출퇴근) 80% / 데스크톱(점심·저녁 자투리) 20% 가정

### 미니멀 원칙 — 시중 앱(말해보카·Anki·Duolingo) 대비 *제외한 것*

| 시중 앱이 가진 것 | gugbab-voca | 근거 |
|---|---|---|
| 음성 인식·발음 평가 | ❌ | PRD 비범위 |
| 챗봇·LLM 실시간 대화 | ❌ | PRD 비범위 |
| 회원가입·로그인·소셜 공유 | ❌ | PRD `인증·계정 = 없음` |
| 멀티 디바이스 동기화 | ❌ | PRD `단일 디바이스 로컬 사용` |
| 콘텐츠 입력 UI (앱 내 단어 추가) | ❌ | PRD `Claude로 정적 JSON 갱신` |
| 결제·구독·광고 | ❌ | PRD 비범위 |
| 푸시 알림·게이미피케이션(레벨업·뱃지) | ❌ | PRD 비범위 |
| 친구·커뮤니티·랭킹 | ❌ | 1인용 |
| 광범위한 통계·차트·히트맵 | P2 (보류) | 출시 시점 미포함 |

### Phase 5 출시 시점 P0 범위 한 줄 요약
> **A1 단어 80 + A1 문장 40을 SM-2 SRS로 플래시카드 학습한다. 5개 화면(홈·모드·학습·요약·설정) + TTS + PWA 오프라인까지 동작한다.** (아키텍처 D2)

## 2. 사용자 시나리오

각 시나리오는 *실제 사용 흐름*을 따라 작성. 페르소나는 모두 박찬호(본인) 동일.

### 시나리오 1 — 첫 진입 (PWA 설치 안내·첫 학습 시작)

| 항목 | 내용 |
|---|---|
| 페르소나 | 박찬호. 출시 직후 처음 앱 URL을 모바일 브라우저로 연다 |
| 상황 | 아무 진도 없음. IndexedDB 비어 있음. 콘텐츠 캐시 없음 |
| 행동 흐름 | 1) 홈 진입 → 콘텐츠 fetch → A1 카드만 활성·나머지 5개는 "준비 중" 비활성<br>2) 우상단 설정 아이콘 옆 "PWA 설치" 토스트 노출 (브라우저 지원 시)<br>3) A1 카드 탭 → 모드 선택 → "단어 학습 (0/80)" 탭<br>4) 첫 카드 표시. *수동 탭 후 자동 재생 활성화* (D1)<br>5) 5장 학습 후 종료 — 세션 요약 |
| 기대 결과 | 첫 5장 학습 완료. 진도가 IndexedDB에 저장됨. PWA 홈 화면 추가 안내 받음 |
| 관련 화면 | `/` → `/level/A1` → `/learn/A1/word` → `/summary` |

### 시나리오 2 — 매일 복습 루틴 (출근길 20장)

| 항목 | 내용 |
|---|---|
| 페르소나 | 박찬호. 출근 지하철 15분. PWA 홈 화면에서 앱 실행 |
| 상황 | A1 단어 30장에 진도 누적. 오늘 due 카드 14장 + 미학습 50장 |
| 행동 흐름 | 1) 홈 → "오늘 복습할 카드: 14장" 요약 노출<br>2) FAB 탭 (마지막 학습 = A1/word로 점프)<br>3) 학습 시작 — 세션 20장 (복습 14 + 신규 6, D 비율 0.3 적용)<br>4) 카드별로 *영어 노출 → TTS 자동 재생 → 속으로 추측 → 탭으로 뒤집기 → 알았음/모르겠음*<br>5) 20장 완료 → 요약 화면 (good 16, again 4)<br>6) "홈으로" 탭 |
| 기대 결과 | 진도가 누적되고 SM-2가 다음 due를 갱신. streak 1일 추가 (Phase 6에서 노출) |
| 관련 화면 | `/` → `/learn/A1/word` (FAB 단축) → `/summary` → `/` |

### 시나리오 3 — 짧은 자투리 시간 (5장만 빠르게)

| 항목 | 내용 |
|---|---|
| 페르소나 | 박찬호. 점심 식후 카페 대기 5분 |
| 상황 | A1 문장 학습 중. 짧게 5장만 |
| 행동 흐름 | 1) 홈 → A1 → "문장 학습 (12/40)" 탭<br>2) 학습 시작 — 기본 20장 세션<br>3) 5장 학습 후 상단 "종료" 아이콘 탭<br>4) 확인 다이얼로그 "5장만 학습한 결과로 종료할까요?" → "종료"<br>5) 요약: 본 카드 5장, 알았음 3, 모르겠음 2 |
| 기대 결과 | 부분 종료 시점까지 본 카드의 SRS 결과는 *저장 유지*. 진행 중이던 세션 자체는 폐기 (PRD 엣지케이스, 아키텍처 7.3) |
| 관련 화면 | `/` → `/level/A1` → `/learn/A1/sentence` → `/summary` |
| 미결 옵션 | 세션 크기 옵션에 5/10도 노출할지 → 검토 포인트 §12 |

### 시나리오 4 — TTS로 발음 듣기 중심 학습

| 항목 | 내용 |
|---|---|
| 페르소나 | 박찬호. 헤드폰 끼고 소리 위주 학습 |
| 상황 | TTS 자동 재생 ON (default) |
| 행동 흐름 | 1) 학습 화면 진입 → 첫 카드는 *수동 탭 1회*로 user gesture 보존 (D1)<br>2) 이후 카드부터 200ms 지연 후 자동 발화<br>3) 화면을 보지 않고 *소리만 듣고 의미 추측*<br>4) 의미 모르면 화면 탭 → 한국어 뒷면 (한국어는 자동 발화 X)<br>5) 알았음/모르겠음 입력 → 다음 카드 자동 발화 시작 |
| 기대 결과 | 시각 의존도 낮춘 청취 학습 가능. 헤드폰 끼고 걸으면서 학습 가능 |
| 관련 화면 | `/learn/A1/word` |
| 주의 | TTS voice 0개·미지원 시 스피커 아이콘 비활성 (PRD 엣지케이스) |

### 시나리오 5 — 설정 변경 (TTS 속도·voice·세션 크기)

| 항목 | 내용 |
|---|---|
| 페르소나 | 박찬호. 기본 1.0배속이 빠르다고 느껴 0.9배속으로 조절 |
| 상황 | 첫 1주 사용 후 학습 패턴 안정화 |
| 행동 흐름 | 1) 홈 우상단 설정 아이콘 탭<br>2) TTS 섹션 — voice 드롭다운으로 다른 voice 선택<br>3) "미리 듣기" 버튼으로 발음 확인<br>4) 재생 속도 슬라이더 0.9배속<br>5) 세션 섹션 — 세션당 카드 수 20 → 30 변경<br>6) 변경값은 *즉시* IndexedDB에 저장 (별도 저장 버튼 X) |
| 기대 결과 | 다음 학습 세션부터 변경된 voice·속도·30장 세션 적용 |
| 관련 화면 | `/settings` |

## 3. 정보 구조 (IA)

### 화면 계층 트리

```
/                           홈 — 레벨 선택
├── /level/:cefr            모드 선택 (단어/문장)
│   └── /learn/:cefr/:mode  학습 세션
│       └── /summary        세션 종료 요약 (메모리 state, 직접 진입 시 홈으로 redirect)
├── /stats                  통계 (P2 — Phase 6)
└── /settings               설정
```

> 라우트 구조는 아키텍처 §3.1 그대로 따른다.

### 화면별 진입·이탈

| 화면 | 진입 경로 | 이탈 경로 |
|---|---|---|
| `/` (홈) | 앱 시작 / 로고 탭 / 요약의 "홈으로" | 레벨 카드 탭·설정 아이콘·통계(P2)·FAB |
| `/level/:cefr` | 홈에서 레벨 카드 탭 | 모드 타일 탭·뒤로가기 |
| `/learn/:cefr/:mode` | 모드 선택 / 홈 FAB / 요약 "한 세션 더" | 종료 버튼 → 요약 / 카드 소진 → 요약 |
| `/summary` | 학습 세션 종료 (자동 또는 수동) | "홈으로"·"한 세션 더 하기"·통계(P2) |
| `/stats` (P2) | 홈 하단 네비 / 요약 CTA | 홈 |
| `/settings` | 홈 헤더 우상단 톱니 아이콘 | 뒤로가기 (홈) |
| `*` 404 | 잘못된 URL | "홈으로" |

### 화면별 목적

| 화면 | 목적 (한 줄) |
|---|---|
| 홈 | 레벨 선택 + 오늘 due 카드 수 알림 + 마지막 학습으로 빠른 진입 |
| 모드 선택 | 레벨 내 단어/문장 콘텐츠 가용성 + 진도 표시 |
| 학습 세션 | 카드 1장씩 표시·자가체크·SRS 갱신 — *집중 모드* |
| 요약 | 세션 결과 시각화 + 다음 액션 분기 |
| 설정 | TTS·세션·진도 데이터 제어 |
| 통계 (P2) | 학습량·정확도·streak |

## 4. 화면별 와이어프레임 + 인터랙션

> 와이어프레임은 *모바일 360~480px 기준*. 데스크톱(≥1024px) 차이는 별도 표기.
> 카드 안 텍스트는 placeholder. 실제 콘텐츠 길이에 맞춰 줄바꿈된다.

| 요소 | 동작 | 결과 |
|---|---|---|
| 레벨 카드 (활성: A1) | 탭 | `/level/A1` 진입 |
| 레벨 카드 (비활성: 준비 중) | 탭 | 토스트 "콘텐츠 준비 중" + 햅틱 (선택) |

| FAB ▶ | 탭 | `lastUsedLevel`·마지막 모드로 `/learn/:cefr/:mode` 직진 |
| FAB (lastUsedLevel = null) | - | FAB 자체를 노출 X |
| 키보드 (데스크톱) | 1~6 키 | 해당 레벨 카드 진입 (A1=1, A2=2, ..., C2=6) |
| 키보드 (데스크톱) | `,` 또는 `s` | 설정 진입 |

#### 상태 분기

| 상태 | UI |
|---|---|
| Loading (콘텐츠/진도 fetch) | 카드 6개 skeleton (회색 박스 + shimmer) |
| Empty (콘텐츠 0개) | 모든 카드 비활성 + 안내 "콘텐츠를 준비 중입니다" |
| Offline | 캐시된 데이터로 동작 + 헤더에 "오프라인" 배지 (회색) |
| Error (JSON 파싱 실패) | 화면 중앙 errorElement: "콘텐츠를 불러올 수 없습니다 [재시도]" |
| IndexedDB 차단 | 모달 (D9) 1회: "설정 변경 후 재시도" / "그래도 진행"(메모리 폴백) |
| 새 SW 발견 | 화면 하단 토스트: "새 콘텐츠가 있습니다 [업데이트]" (D4) |

#### PRD 수용기준 매핑
- **AC-1 (레벨 선택)** → 레벨 카드 탭 → 모드 선택 진입 / 0개면 비활성
- **AC-7 (PWA 오프라인)** → 캐시된 콘텐츠로 정상 동작

|---|---|---|
| 카드 본체 | 탭 → 뒷면 뒤집기 | 탭 → 앞면 뒤집기 (드물지만 허용) |

#### TTS 자동 재생 정책 (D1 반영)

| 시점 | 동작 |
|---|---|
| 세션 *첫 카드* | 사용자가 한 번 탭하기 전까지 자동 재생 X (iOS Safari user gesture 보존) |
| 첫 카드 첫 탭 이후 | `ttsAutoPlay = true`이면 자동 재생 활성 |
| 두 번째 카드부터 | 카드 표시 후 200ms 지연 → 자동 발화 |
| 카드 뒷면 진입 | 한국어는 자동 발화 X (영어 학습 목적) |
| 카드 이동 시 | 이전 utterance `cancel()` 즉시 호출 |

#### 상태 분기

| 상태 | UI |
|---|---|
| Loading (queue 합성 중) | 카드 skeleton + ProgressBar 0/N |
| Empty (due+new=0) | `/summary?empty=true` redirect → "오늘 학습할 카드가 없습니다" |

| TTS voice 0개 | 동일 |
| 자가체크 더블 클릭 | 첫 입력 처리 후 200ms 버튼 disabled (PRD 엣지케이스) |
| 백그라운드 진입 (모바일) | TTS `cancel()` — 복귀 시 재탭 필요 |
| `prefers-reduced-motion` | 카드 뒤집기 애니메이션 → 단순 fade 0.15s |
| 자동 뒤집기 ON (D8) | 카드 표시 후 N초 뒤 *자동으로 뒷면*. 자가체크는 *수동* 유지 — 입력 없으면 다음 카드로 진행 X |

#### PRD 수용기준 매핑
- **AC-2 (학습 세션 시작)** → queue 합성 N장 + due 우선·신규 보충 + 셔플
- **AC-3 (카드 뒤집기·자가체크)** → 탭/Space 뒤집기, 알았음/모르겠음 시 IndexedDB 갱신 + 진도 +1, 앞면일 때 자가체크 버튼 비활성
- **AC-4 (TTS 듣기)** → 스피커 탭 시 영어 발화, voice 0개 시 비활성

## 9. 접근성 (WCAG 2.1 AA)

### 9.1 색상 대비

| 요소 | 기준 | 검증 |
|---|---|---|
| 본문 텍스트 (16px) | 4.5:1 | `--text-primary` vs `--bg-default` |
| 큰 텍스트 (≥24px 또는 ≥18.66px bold) | 3:1 | h2·h3 |
| UI 요소 (버튼·아이콘) | 3:1 | primary·error 컬러 |
| 비활성 요소 | 기준 면제 (WCAG 1.4.3 예외) | "준비 중" 카드 |
| **색만으로 정보 전달 금지** | 알았음/모르겠음은 *색 + 텍스트 + 아이콘* (←→) 3중 표기 | PRD 요구사항 |

### 9.2 키보드 접근성

| 액션 | 키 (데스크톱 only — D10) |
|---|---|
| 카드 뒤집기 | Space |
| 모르겠음 | ← |
| 알았음 | → |
| TTS 재생 | P |
| 화면 이동 | Tab / Shift+Tab |
| 버튼 활성 | Enter |
| 다이얼로그 닫기 | Esc |
| 홈에서 레벨 점프 | 1~6 |

- 모든 인터랙티브 요소는 Tab으로 도달 가능
- focus 링: 2px solid `--mui-palette-primary-main` + offset 2px (`outline-offset`)
- focus 가시성: 배경 대비 3:1 이상

### 9.3 스크린리더 지원

| 요소 | 처리 |
|---|---|
| Flashcard 앞면→뒷면 전환 | `aria-live="polite"` 영역에 "한국어: {korean}" 알림 |
| 진도 변화 | `aria-live="polite"` 영역에 "{current}장 중 {n}장 완료" |
| 자가체크 결과 | `aria-live="polite"`로 "다음 카드: {english}" 알림 (선택, 너무 시끄러우면 제외) |
| 스피커 버튼 | `aria-label="영어 발음 듣기"` |
| LevelCard | `aria-label="A1 — 기초 일상, 80개 중 30개 학습, 오늘 복습 14개"` |
| LevelCard 비활성 | `aria-disabled="true"` + `aria-label="A1 — 콘텐츠 준비 중"` |
| ProgressBar | `role="progressbar"` + `aria-valuenow / aria-valuemax / aria-valuetext` |
| ConfirmDialog | `role="dialog"` + `aria-labelledby` + `aria-describedby` + 첫 진입 시 포커스 trap |

### 9.4 모션·애니메이션

| 요소 | 기본 | `prefers-reduced-motion: reduce` |
|---|---|---|
| 카드 뒤집기 | 0.4s 3D rotateY | 0.15s opacity fade |
| 다음 카드 전환 | 0.2s fade | 즉시 표시 |
| FAB hover lift | 0.2s transform | 변화 없음 |
| Toast slide-in | 0.3s | 즉시 표시 |
| Skeleton shimmer | 무한 반복 | 정적 회색 |

### 9.5 TTS 시각 보완 — 본인 1인용 예외

> WCAG는 음성에 자막·캡션 권장하나, 본 앱은 *영어 발음 학습이 목적*이므로 캡션 미제공.
> 텍스트(영어)가 항상 화면에 존재 → 시각 보완 효과 자체는 충족.

## 10. 톤·무드보드

### 10.1 전체 톤
- **미니멀·집중·차분** — 학습 방해 최소화
- 시각적 노이즈 최소: 그라데이션·일러스트·이모지(필수 외) 제외
- 중요한 액션 1~2개만 강조 (Primary 버튼은 화면당 1개)
- 학습 화면은 "독서 앱" 같은 정적 분위기

### 10.2 컬러 팔레트 방향
- **흰 배경 + Primary 1색 + 회색 그라데이션**
- Primary: 차분한 블루 계열 (예: `#2563EB`) — 집중·신뢰
- Error: 부드러운 레드 (예: `#DC2626`) — "모르겠음"의 부정적 인지 부담 완화
- 그라데이션·네온·메탈릭 효과 X
- 다크모드 (P2): 순수 검정 X. `#121212` 기반 — OLED 친화 + 눈 부담 완화

### 10.3 폰트
- **시스템 폰트 우선**: `-apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, "Helvetica Neue", Arial, sans-serif`
- 한글 폴백: Pretendard 또는 시스템 한글
- 영어 카드 텍스트는 *높은 가독성 sans-serif* (시스템 폰트로 충분)
- 외부 폰트 다운로드 X — PWA 첫 진입 LCP 보호

### 10.4 애니메이션 가이드라인

| 애니메이션 | duration | easing |
|---|---|---|
| 카드 뒤집기 | 0.4s | ease-out |
| 다음 카드 fade | 0.2s | ease-out |
| 버튼 press scale | 0.1s | ease-out (scale 0.98) |
| FAB lift on hover | 0.2s | ease-out |
| Toast slide-in | 0.3s | cubic-bezier(0.4, 0, 0.2, 1) |
| Skeleton shimmer | 1.5s linear infinite | - |

> `prefers-reduced-motion: reduce` 시 위 모든 transition은 0.01s로 단축 (즉시 표시와 동등).

### 10.5 아이콘 가이드라인

  - 설정: `Settings` (톱니)
  - 뒤로: `ArrowBack`
  - 종료: `Close`
  - 일시중지: `Pause`

     A) 2x3 (제안 — 모바일 기본) ✓
     B) 6x1 (세로 스크롤로 한 줄씩)
     C) 1x6 (가로 스크롤 캐러셀)

     B) 카드 바로 아래 (스크롤 필요할 수 있음)
     C) 카드 좌우 (스와이프 + 버튼 둘 다)

□ 4. 세션 크기 default = 20장 (PRD)
     - 변경 의견? 10장이 더 자투리 시간 친화적일 수 있음

□ 6. 다크모드 P2 (Phase 6) — Phase 5(출시) 전에 당겨올 가치?

□ 7. 컴포넌트 인벤토리에 추가/제외할 것

     - 학습 앱 전반의 "재미" 요소 (이모지·일러스트) 추가 의견

□ 9. 진도 초기화 위치
     - 설정 화면에 두는 게 맞나, 더 깊은 곳(예: "고급 설정")에 숨길지

□ 10. 키보드 단축키 안내 표시 위치
     - 학습 화면 하단 고정 vs Esc 누르면 cheat sheet 모달 vs 안 보임

|---|---|
