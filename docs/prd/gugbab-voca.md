# gugbab-voca — 기능 명세서 (PRD)

## 개요

- **프로젝트명(가칭)**: gugbab-voca
- **한 줄 설명**: 영어 회화에서 자주 쓰는 단어·문장을 CEFR 6단계(A1~C2) 기준으로 학습하는 1인용 웹/PWA 앱
- **목적**: 본인이 직접 큐레이션한 영어 회화 단어·문장을 SRS(간격 반복) 기반 플래시카드로 효율적으로 암기·복습한다
- **대상 사용자**: 본인 1명 (단일 디바이스 사용 가정, 다중 사용자/계정 개념 없음)
- **작성일**: 2026-05-07
- **성공 지표 (KPI)**
  1. **꾸준한 학습 지속률**: 주당 학습 세션 3회 이상 유지 (개인 KPI)
  2. **카드 정착률**: SRS 상태가 "review" 단계(첫 통과 이후)로 전환된 카드 비율 — 콘텐츠 추가 후 30일 시점 60% 이상
  3. **세션 완료율**: 시작한 세션을 끝까지 진행한 비율 80% 이상 (이탈 적은 UX의 지표)

| 음성 인식·발음 평가·실시간 대화 | **모두 비범위** |

## 사용자 스토리

핵심(P0)·보조(P1)·있으면 좋음(P2) 우선순위로 구분한다.

### P0 — 핵심

1. **레벨 선택**
   - AS A 학습자
   - I WANT TO 홈 화면에서 CEFR 6단계 중 학습할 레벨을 선택한다
   - SO THAT 내 수준에 맞는 콘텐츠로 학습할 수 있다

2. **콘텐츠 타입 선택 (단어/문장)**
   - AS A 학습자
   - I WANT TO 레벨 진입 후 단어 학습 또는 문장 학습 중 하나를 선택한다
   - SO THAT 어휘와 표현 학습을 분리해 집중할 수 있다

3. **학습 모드 선택 (4종)**
   - AS A 학습자
   - I WANT TO 콘텐츠 타입 진입 후 *플래시카드 / 리콜 / 클로즈 / 단어장* 중 하나를 선택한다
   - SO THAT 학습 단계·집중 영역에 맞춘 다양한 방식으로 학습한다
   - 비고: 단어 콘텐츠에는 클로즈 모드 미적용. 문장 콘텐츠는 4종 모두 적용

4. **플래시카드 학습 (Mode A)**
   - AS A 학습자
   - I WANT TO 영어 면을 보고 한국어 면으로 뒤집은 후 "알았음/모르겠음"을 입력한다
   - SO THAT 자가 인지 과정을 통해 기억을 강화한다

5. **리콜 학습 (Mode B, 한→영 입력)**
   - AS A 학습자
   - I WANT TO 한국어 뜻을 보고 영어 단어/문장을 직접 키보드로 입력한다
   - SO THAT 능동적 회상(active recall)을 통해 더 깊은 학습 효과를 얻는다
   - 자동 채점 후: 정답이면 다음 카드, 오답이면 "정답 보기" 버튼 → 정답 노출 후 진행

6. **클로즈 학습 (Mode C, 빈칸 채우기 — 문장만)**
   - AS A 학습자
   - I WANT TO 영어 문장의 빈칸 위치에 들어갈 단어를 입력한다
   - SO THAT 문맥 속에서 단어를 활용하는 능력을 강화한다
   - 빈칸 위치는 콘텐츠 JSON의 `cloze` 필드에서 명시 (`{english:"I {go}...", cloze:["go"]}`)

7. **단어장 (Mode D, 조회 + 마킹)**
   - AS A 학습자
   - I WANT TO CEFR 레벨별 전체 단어/문장을 리스트로 본다. 검색·학습 상태(X/중/마스터)를 확인할 수 있다
   - I WANT TO 각 카드를 "안다(known)" 또는 "모른다(unknown)"로 마킹하여 학습 우선순위를 조정할 수 있다
   - SO THAT 이미 아는 단어는 학습 풀에서 빈도를 낮추고, 모르는 단어를 우선 학습할 수 있다

8. **TTS 듣기**
   - AS A 학습자
   - I WANT TO 카드의 영어 텍스트를 음성으로 들을 수 있다 (모든 학습 모드 + 단어장 상세)
   - SO THAT 발음·억양 감각을 함께 익힌다

9. **세션 종료 요약**
   - AS A 학습자
   - I WANT TO 한 세션이 끝나면 본 카드 수, 정답률(또는 알았음 비율)을 확인한다
   - SO THAT 학습량을 인지하고 다음 세션으로 동기를 이어간다

### P1 — 보조

6. **SRS 기반 자동 복습**
   - AS A 학습자
   - I WANT TO 이전에 "모르겠음"을 누른 카드가 적절한 시점에 다시 등장한다
   - SO THAT 잊을 만할 때 자동으로 복습할 수 있다

7. **설정 변경**
   - AS A 학습자
   - I WANT TO TTS 음성·세션당 카드 수·플래시카드 자동 뒤집기 등을 설정할 수 있다
   - SO THAT 내 학습 스타일에 맞춘다

### P2 — 있으면 좋음

8. **간단 통계**
   - AS A 학습자
   - I WANT TO 레벨/모드별 학습한 카드 수, 정답률, 연속 학습일을 본다
   - SO THAT 진척을 시각적으로 확인하고 동기 부여를 받는다

9. **PWA 오프라인 학습**
   - AS A 학습자
   - I WANT TO 인터넷이 없을 때도 다운로드된 콘텐츠로 학습할 수 있다
   - SO THAT 어디서나 짬짬이 학습한다

## 화면 구성 및 사용자 플로우

### 전체 플로우

```
[홈/레벨 선택]
      │
      ▼ (레벨 카드 탭)
[모드 선택 (단어 / 문장)]
      │
      ▼ (모드 탭)
[학습 중 (플래시카드)]
      │  ├─ 뒤집기 → 자가체크(알았음/모르겠음) → 다음 카드
      │  └─ TTS 재생 / 진도 표시 / 일시중지
      ▼ (세션 카드 소진 또는 사용자 종료)
[세션 종료 요약]
      │
      ▼ (홈으로 / 다시 시작 / 통계 보기)
[홈] 또는 [통계] 또는 [설정]
```

---

### 1) 홈 / 레벨 선택 화면

- **진입 경로**: 앱 시작 시 기본 진입점 (`/`)
- **구성 요소**
  - 앱 헤더 (로고/타이틀 "gugbab-voca", 우상단 설정 아이콘)
  - CEFR 레벨 카드 6개 (A1·A2·B1·B2·C1·C2)
    - 각 카드: 레벨 라벨, 짧은 설명(예: "A1 — 기초 일상"), 진행률 바(학습한 카드 / 전체 카드), 오늘 due 카드 수 배지
  - 하단 네비: 홈 / 통계(P2) / 설정
- **사용자 액션**
  - 레벨 카드 탭 → 모드 선택 화면 진입
  - 설정 아이콘 탭 → 설정 화면 진입
- **상태**
  - 로딩: 레벨 카드 스켈레톤
  - 빈 상태: 해당 레벨에 콘텐츠가 0개면 카드 비활성 + "콘텐츠 준비 중" 안내
  - 에러: JSON 로드 실패 시 "콘텐츠를 불러올 수 없습니다 — 새로고침" 토스트

---

### 2) 모드 선택 화면

- **진입 경로**: 홈에서 레벨 카드 탭
- **구성 요소**
  - 상단: 선택한 레벨 라벨 + 뒤로가기
  - 큰 타일 2개: "단어 학습", "문장 학습" — 각 타일에 해당 모드의 due 카드 수 / 전체 카드 수 표시
- **사용자 액션**
  - 타일 탭 → 학습 중 화면 진입
  - 뒤로가기 → 홈
- **상태**
  - 빈 상태: 단어/문장 어느 한쪽이 0개면 해당 타일 비활성

---

### 3) 학습 중 (플래시카드) 화면

- **진입 경로**: 모드 선택에서 단어/문장 타일 탭
- **구성 요소**
  - 상단: 진도 표시 (예: `7 / 20`), 일시중지/종료 버튼, 현재 모드·레벨 라벨
  - 중앙: 플래시카드
    - 앞면: 영어 텍스트 (단어 또는 문장), TTS 재생 버튼(스피커 아이콘), 단어 모드라면 품사(part of speech)
    - 뒷면: 한국어 뜻 (+ 단어 모드는 짧은 예문 한 줄, 문장 모드는 직역+의역 둘 다 가능)
  - 하단:
    - 카드가 앞면일 때 → "뒤집기" 버튼
    - 카드가 뒷면일 때 → "모르겠음" / "알았음" 버튼 2개
  - (선택) 카드를 가로로 스와이프해도 동일 액션 트리거
- **사용자 액션**
  - 카드 탭 또는 "뒤집기" 버튼 → 카드 뒤집기 (앞 ↔ 뒤)
  - "알았음" → 다음 카드, SRS 입력 "good"
  - "모르겠음" → 다음 카드, SRS 입력 "again"
  - TTS 버튼 → `speechSynthesis`로 영어 텍스트 발화
  - 일시중지/종료 → 세션 종료 요약 화면 (현재까지 본 결과로)
  - 키보드 단축키: Space=뒤집기, ←=모르겠음, →=알았음, P=재생
- **상태**
  - 로딩: 카드 스켈레톤 (첫 카드 로드 중)
  - 빈 상태: due 카드 + 신규 카드 모두 0이면 즉시 종료 요약 "오늘 학습할 카드가 없습니다"
  - 에러: TTS 미지원 시 스피커 아이콘 비활성 + 툴팁 "이 브라우저는 음성 재생을 지원하지 않습니다"

---

### 4) 세션 종료 요약 화면

- **진입 경로**: 학습 중 → 세션 카드 소진 또는 사용자 종료
- **구성 요소**
  - 큰 텍스트: "오늘 학습 완료!" / "이번 세션 결과"
  - 통계 카드
    - 본 카드 수
    - 알았음/모르겠음 카운트와 비율
    - 다음 복습 예정 카드 수 (SRS 결과 기반)
    - (P2) 연속 학습일(streak) 표시
  - CTA 버튼
    - "한 세션 더 하기" → 같은 레벨·모드로 새 세션 시작
    - "홈으로" → 홈
    - "통계 보기"(P2) → 통계 화면
- **상태**
  - 일반 상태만 존재. 데이터는 직전 세션 메모리에 유지

---

### 5) 통계 / 이력 화면 (P2)

- **진입 경로**: 홈 하단 네비 또는 세션 종료 요약 CTA
- **구성 요소**
  - 전체 누적: 총 학습 카드 수, 평균 정답률, 연속 학습일
  - 레벨×모드 매트릭스(6×2): 카드별 학습 진행률
  - 최근 7일/30일 학습량 그래프(P2)
- **사용자 액션**: 데이터 보기 전용 (편집 없음)
- **상태**
  - 빈 상태: 학습 이력 없으면 "아직 학습 기록이 없어요" 안내

---

### 6) 설정 화면

- **진입 경로**: 홈 헤더 우상단 톱니 아이콘
- **구성 요소**
  - **TTS 설정**
    - 음성(voice) 선택 드롭다운 (브라우저가 제공하는 영어권 voice 목록)
    - 재생 속도(0.5~1.5x), 미리 듣기 버튼
    - 카드 표시 시 자동 재생 on/off
  - **세션 설정**
    - 세션당 카드 수 (기본 20, 옵션: 10/20/30/50)
    - 신규 카드 비율 (기본 30%, 0~100%)
    - 카드 자동 뒤집기 시간 (off / 3s / 5s / 10s)
  - **학습 데이터**
    - 진도 초기화(레벨/모드별, 또는 전체) — 확인 다이얼로그 필수
    - 진도 JSON 내보내기/가져오기 (P2, 디바이스 이전용)
  - **앱 정보**
    - 버전, 콘텐츠 빌드 일시, PWA 설치 안내
- **사용자 액션**: 변경 즉시 IndexedDB(또는 localStorage)에 저장
- **상태**
  - TTS voice 목록이 비어있으면 "사용 가능한 음성이 없습니다" 안내

## 수용 기준 (Given-When-Then)

### AC-1: 레벨 선택

```
GIVEN 콘텐츠 JSON이 정상 로드되고 A1 레벨에 단어 100개·문장 50개가 있다
WHEN  사용자가 홈에서 A1 카드를 탭한다
THEN  모드 선택 화면이 열리고 "단어 학습 (xx/100)" "문장 학습 (xx/50)"이 표시된다
```

```
GIVEN A2 레벨에 단어 0개·문장 0개가 있다
WHEN  사용자가 홈에서 A2 카드 영역을 본다
```

### A. 정적 콘텐츠 JSON

콘텐츠는 빌드 시점에 정적 JSON 파일로 번들된다. 레벨별로 분할하여 초기 로드 비용을 최소화한다.

#### 파일 배치

```
public/data/
  words/
    a1.json
    a2.json
    b1.json
    b2.json
    c1.json
    c2.json
  sentences/
    a1.json
    ...
  manifest.json   // 전체 콘텐츠 빌드 메타
```

#### `WordEntry` 스키마

```jsonc
{
  "id": "w_a1_001",            // 고유 ID, "w_{level}_{seq}" 패턴 권장 (불변)
  "level": "A1",                // CEFR enum: A1|A2|B1|B2|C1|C2
  "english": "hello",           // 영어 표제어
  "korean": "안녕하세요",       // 한국어 뜻 (대표 1개)
  "partOfSpeech": "interjection", // enum: noun|verb|adjective|adverb|pronoun|preposition|conjunction|interjection|determiner|other
  "ipa": "/həˈloʊ/",            // (선택) IPA 발음 표기
  "examples": [                 // (선택) 짧은 예문 배열
    { "en": "Hello, how are you?", "ko": "안녕, 어떻게 지내?" }
  ],
  "tags": ["greeting"]          // (선택) 주제 태그
}
```

#### `SentenceEntry` 스키마

```jsonc
{
  "id": "s_a1_001",
  "level": "A1",
  "english": "I {go} to school every day.",  // 클로즈 빈칸은 {word} 마커로 표기
  "korean": "나는 매일 학교에 간다.",   // 대표 의역
  "literal": "나는 학교에 매일 간다.", // (선택) 직역
  "cloze": ["go"],                // 빈칸 정답 단어 배열. 마커 순서와 일치. 클로즈 모드 전용
  "context": "daily-routine",          // (선택) 상황 태그
  "tags": ["daily", "verb"]   // (선택)
}
```

> 클로즈 빈칸 규약 (M2):
> - `english` 필드의 `{단어}` 마커가 빈칸 위치
> - 빈칸 정답은 `cloze` 배열의 동일 인덱스 단어
> - 빈칸 1개 이상 가능. 클로즈 모드에서만 사용
> - 일반 표시(플래시카드/리콜)에서는 마커를 단어로 치환하여 출력 (`"I go to school every day."`)

#### `manifest.json` 스키마

```jsonc
{
  "buildAt": "2026-05-07T08:00:00Z",
  "schemaVersion": 1,
  "counts": {
    "words":     { "A1": 100, "A2": 100, "B1": 100, "B2": 80, "C1": 60, "C2": 40 },
    "sentences": { "A1":  50, "A2":  50, "B1":  50, "B2": 40, "C1": 30, "C2": 20 }
  }
}
```

| 레벨 | 단어 (최소 권장) | 문장 (최소 권장) |

| A1 | 80 | 40 |

### C. 콘텐츠 데이터 → 진도 데이터 매칭 규칙

- `cardId` = `WordEntry.id` 또는 `SentenceEntry.id`로 1:1 매칭
- 콘텐츠 JSON에서 `id`가 변경되면 진도가 끊김 → **`id`는 한 번 발급되면 절대 바꾸지 않는다** (Claude로 갱신 시 규칙 명시)

## SRS 입력 처리 규칙 (추상)

> 알고리즘은 **SM-2** 직접 구현. 본 PRD는 **외부 동작 규칙**만 정의한다.

### 모드별 SRS rating 결정

| 학습 모드 | rating='good' 조건 | rating='again' 조건 |
|---|---|---|
| 플래시카드 | "알았음" 버튼 탭 | "모르겠음" 버튼 탭 |
| 리콜 | 입력값과 정답 매칭 성공 | 매칭 실패 후 "정답 보기" |
| 클로즈 | 모든 빈칸이 정답 | 하나라도 오답 후 "정답 보기" |

### 입력 매칭 정책 (M3, 리콜·클로즈 공통)

```
function normalize(s: string): string {
  return s.trim()
          .toLowerCase()
          .replace(/[.,?!;:'"]/g, '')   // 기본 구두점 제거
}

function isCorrect(input: string, expected: string): boolean {
  return normalize(input) === normalize(expected)
}
```

- 대소문자 무시 (`Apple` ≅ `apple`)
- 앞뒤 공백 trim
- 기본 구두점 (`.,?!;:'""`) 제거 — `"Apple."` ≅ `"apple"`
- **오타는 오답** — Levenshtein 같은 관용 매칭은 도입 X (학습 효과 우선)
- 빈칸 여러 개(클로즈)는 모든 빈칸이 정답이어야 `good`

### "정답 보기" UX 규칙

```
오답 입력 → "다시 시도" / "정답 보기" 두 버튼 노출
  - "다시 시도": 입력창 클리어, 같은 카드 유지. SRS rating 미확정
  - "정답 보기": 정답 노출 (입력값과 비교 강조), SRS rating='again' 확정,
                "다음 카드" 버튼 활성화
```

> 주의: "다시 시도"는 *동일 카드 동일 시도*로 카운트. SRS는 마지막 결과만 반영.
> 단, 한 카드에서 3회 연속 오답이면 자동으로 "정답 보기" 모드 전환 (사용자 좌절 방지).

### 자가체크 입력 (플래시카드 모드)

| 입력 | 의미 | 다음 노출 효과 |
|------|------|----------------|
| **알았음** (`good`) | 기억함 | 간격이 늘어나고 `dueAt`이 미래로 이동. `reps += 1` |
| **모르겠음** (`again`) | 잊었음 | 간격이 짧아지고(보통 즉시~10분), state는 `learning` 또는 `relearning`으로. `lapses += 1` |

### 상태 전이

```
new ──good──▶ learning ──good──▶ review
                  │
                  └─again──▶ learning (간격 리셋)

review ──good──▶ review (간격 증가)
       └─again──▶ relearning ──good──▶ review (간격 단축 후 재진입)
```

### 보장 사항

- "알았음"을 연속해서 누를수록 다음 `dueAt` 간격은 단조 증가한다
- "모르겠음" 직후 카드는 같은 세션 내 또는 짧은 시간 내 다시 등장한다
- 어떤 입력이든 `lastReviewedAt`이 갱신되고 진도가 IndexedDB에 즉시 저장된다 (앱 종료에도 유실 없음)

## 셔플 vs SRS 우선순위 정책

### 세션 카드 구성 알고리즘

세션당 카드 수 `N` (기본 20), 신규 카드 비율 `R` (기본 0.3), 학습 모드 `studyMode` 라 하자.

```
1. (cardType, level, studyMode) 조건의 cardProgress에서:
   - dueAt <= now 인 review/learning/relearning 카드 후보 = D
   - state == "new" 인 카드 후보 = NEW_PROGRESS
2. 콘텐츠 JSON에서 cardProgress 미존재 카드 후보 = NEW_RAW
   - NEW = NEW_PROGRESS ∪ NEW_RAW

3. 마킹 가중치 적용 (M5):
   - NEW_unknown  = NEW.filter(c => c.userMark === 'unknown')
   - NEW_unmarked = NEW.filter(c => c.userMark == null)
   - NEW_known    = NEW.filter(c => c.userMark === 'known')

4. 목표 카운트:
   - dueCount   = min(|D|,   floor(N * (1 - R)))   ← 복습 우선
   - newTarget  = N - dueCount

5. 신규 풀 가중치 분배 (newTarget 채우기):
   - newUnknownCount  = round(newTarget * 0.70)  ← 70%
   - newUnmarkedCount = round(newTarget * 0.25)  ← 25%
   - newKnownCount    = round(newTarget * 0.05)  ← 5%
   - 부족분(예: NEW_unknown이 0)은 우선순위 ↓ 풀에서 보충
     unknown 부족 → unmarked → known 순으로 channel-fallback

6. due 카드: dueAt 오름차순(가장 오래 묵은 것부터) → 동시 due면 셔플
7. 각 신규 채널: 셔플 후 head N개 추출
8. 최종: due + new를 라운드로빈 인터리빙 (단조로움 회피)
```

### 보장 사항

- 처음 학습 시작(진도 0)에는 모든 카드가 신규 → unknown 우선·known 거의 안 나옴
- due 카드가 `N * (1-R)`를 초과하면 신규는 한 장도 안 나올 수 있다 (복습 우선이 의도된 동작)
- 모든 카드가 review·미래 due면 "오늘 학습할 카드가 없습니다" 빈 상태 노출
- known 마킹 카드도 5% 확률로 섞여 등장 → 자기평가 검증 기회 제공 (M5 의도)
- userMark 변경은 *다음 세션부터* 반영 (현재 진행 중 큐는 영향 X)

### 마킹 → 첫 학습 SRS 초기값 (M6)

신규 카드(`state === 'new'` 또는 cardProgress 미존재)가 처음 학습되어 SRS 진도 row가 생성될 때, `userMark`에 따라 초기값을 다르게 설정:

| userMark | 초기 EF | 초기 intervalDays | 의도 |
|---|---|---|---|
| `'known'` | 3.0 | 6 | 사용자가 안다고 했으므로 신뢰. 첫 학습 후 6일 뒤 검증 |
| `'unknown'` | 2.0 | 1 | 사용자가 모른다고 했으므로 빨리·자주 검증 |
| `null` | 2.5 | 1 | SM-2 표준 |

> 주의: 자가체크/자동채점 결과(rating)도 평소처럼 SM-2에 입력된다. 마킹은 *초기값만* 결정하고, 이후 review 결과에 따라 정상 EF 진화.

## 엣지 케이스 & 예외 처리

| 케이스 | 처리 |
|--------|------|
| **레벨에 콘텐츠 0개** | 홈에서 해당 레벨 카드 비활성, "콘텐츠 준비 중" 표기. 모드 선택 진입 차단 |
| **단어 또는 문장만 0개** | 모드 선택 화면에서 해당 모드 타일 비활성 |
| **모든 카드 SRS 완료(미래 due)** | 학습 시작 시 즉시 세션 종료 화면으로, "오늘 학습할 카드 없음 — 다음 due: YYYY-MM-DD HH:mm" 안내 |
| **TTS voice 미지원** | 스피커 아이콘 비활성 + 안내 툴팁. 학습 동작은 정상 |
| **TTS voice 0개** | 위와 동일 처리. 설정 화면에서도 "사용 가능한 음성 없음" 표기 |
| **IndexedDB 차단/사용 불가** (시크릿 모드 등) | 첫 진입 시 모달로 안내 "진도 저장이 차단됨 — 일반 창에서 사용해주세요". 학습은 메모리 한정으로 가능하되 종료 시 손실 |
| **PWA 오프라인** | Service Worker가 콘텐츠 JSON·앱 셸 캐시. 오프라인에서도 학습/진도 저장 가능. 새 콘텐츠 빌드는 다음 온라인 접속 시 갱신 |
| **콘텐츠 JSON 파싱 실패** | 글로벌 에러 바운더리에서 "콘텐츠를 불러올 수 없습니다" + 재시도 버튼 |
| **콘텐츠 갱신으로 카드 ID 사라짐** | 진도 레코드는 보존(또는 설정으로 정리). 세션 구성 시 무시 |
| **콘텐츠 갱신으로 카드 추가** | 자동으로 `state="new"`로 인지되어 신규 풀에 합류 |
| **세션 도중 새로고침/탭 닫음** | 현재까지 저장된 카드별 SRS 결과는 유지. 진행 중이던 세션 자체는 폐기 |
| **시계 변경/시간대 변경** | `dueAt`은 epoch ms 절대시각이므로 영향 없음. 화면의 "다음 복습일" 표시는 디바이스 로컬 타임존 기준 |
| **매우 큰 콘텐츠** (5000+ 카드) | 레벨별 JSON 분할 + 학습 시점에만 해당 레벨 로드. IndexedDB 인덱스로 due 조회 O(log n) 보장 |
| **자가체크 더블 클릭/연타** | 첫 입력 처리 후 다음 카드까지 버튼 disable (200ms 디바운스) |
| **TTS 발화 중 다음 카드 이동** | 이전 발화 즉시 `cancel()` 후 다음 카드 발화(자동 재생 켜진 경우) |
| **모바일 백그라운드 진입** | TTS 일시중지(`speechSynthesis.pause()`), 복귀 시 사용자가 다시 재생 |
| **키보드 사용자** | 모든 액션은 Tab/Enter/Space로 접근 가능, 화살표 단축키로 자가체크 |

- 다중 사용자·계정·동기화 (다른 디바이스 간)

## 기술 참고 (제안)

> 구현 단계에서 확정. PRD 종속 아님.

- **빌드**: Vite + React + TypeScript
- **PWA**: `vite-plugin-pwa` (Workbox 기반)
- **UI**: `@gugbab-ui/styled-mui` (MUI 감성 디자인 시스템)
- **상태 관리**: 로컬 UI는 useState/useReducer, 전역 상태(설정·세션)는 Zustand
- **데이터 페칭**: 정적 JSON `fetch`, 캐싱은 SW에 위임
- **IndexedDB**: Dexie.js
- **SRS 라이브러리**:
  - SM-2: 직접 구현(코드 양 적음, 검증된 공식)
  - FSRS: `ts-fsrs` 등 활용 (더 정교, 학습 곡선 데이터에 적응적)
- **TTS**: 브라우저 내장 `window.speechSynthesis` + voice 가용성 폴링
- **테스트**: Vitest + Testing Library, Playwright(E2E 핵심 플로우 1~2개)
- **호스팅**: Vercel 또는 Netlify (정적)
- **콘텐츠 갱신 워크플로우**: 본인 → Claude에 프롬프트 → 갱신된 JSON 받기 → 레포에 커밋 → 자동 배포

## 미결 사항

1. **SRS 알고리즘 선택**: SM-2(단순·검증) vs FSRS(정교·학습 적응형) — 1차 MVP는 SM-2로 시작 후 FSRS로 점진 교체 권장. 결정 필요.
2. **콘텐츠 데이터셋**: NGSL / Oxford 3000 / English Profile 중 무엇을 시드로 쓸지 — 라이선스 확인 필요.
3. **진도 데이터 백업**: 기기 변경/브라우저 데이터 삭제 대비, JSON 내보내기/가져오기를 P1로 올릴지 P2로 둘지.
4. **연속 학습일(streak) 정의**: 자정 기준인가 24시간 롤링인가, 0장 학습일은 깨지는가.
5. **자동 뒤집기 모드의 자가체크 처리**: 자동 뒤집기 후에는 "알았음/모르겠음" 입력을 어떻게 받을지 (타임아웃 후 자동 again 처리 여부 등).
6. **PWA 업데이트 알림**: 새 콘텐츠 빌드가 배포됐을 때 사용자에게 알림 표시 여부와 방식.
7. **다크 모드**: `@gugbab-ui/styled-mui`에 다크 토큰이 있다면 지원. 현재는 미정.
