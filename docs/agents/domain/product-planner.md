# product-planner

## 개요

- **역할**: 기능 아이디어나 요구사항을 입력받아 구조화된 기능 명세서(PRD)를 작성하는 기획 에이전트. 사용자 스토리, 수용 기준, 화면 흐름, 데이터 요구사항, 엣지 케이스를 체계적으로 정리한다.
- **모델**: opus
- **도구**: Read, Write, Glob, Grep
- **카테고리**: domain

## 사용 시점

- 새로운 기능 아이디어를 개발 가능한 명세서로 구체화할 때
- 사용자 스토리와 수용 기준(Given-When-Then)을 체계적으로 작성할 때
- 화면 흐름과 구성 요소를 정의할 때
- 엣지 케이스와 예외 상황을 빠짐없이 도출해야 할 때
- 기존 코드베이스를 분석해 현재 구조에 맞는 기획을 해야 할 때

## 사용 예시

- "장바구니 기능 기획해줘"
- "이 요구사항으로 PRD 작성해줘"
- "알림 센터 기능 명세서 만들어줘"

## 입력/출력

- **입력**: 기능 아이디어 또는 요구사항 설명, 프로젝트 경로 (선택)
- **출력**: 마크다운 형식의 기능 명세서(PRD) -- 개요, 사용자 스토리(P0/P1/P2 우선순위), 화면 흐름, 수용 기준(Given-When-Then), 데이터 요구사항, 엣지 케이스, 미결 사항 포함. 요청 시 `docs/prd/` 경로에 파일로 저장

## 관련 에이전트

- **business-domain-analyst** (domain) -- DDD 관점의 도메인 모델 도출. 기획과 도메인 모델링이 동시에 필요할 때 협업
- **codebase-domain-analyst** (domain) -- 기존 코드베이스 역분석. 기획 시 현재 시스템 구조를 파악해야 할 때 활용
- **planner** (meta) -- 기획 결과를 실행 단계로 분해할 때 활용
- **frontend-architect** (frontend) -- 화면 흐름 기획 후 프론트엔드 아키텍처 설계로 이어질 때
