# planner

## 개요

- **역할**: 복잡한 작업 요청을 2~5분 단위의 구체적인 단계로 분해하고, 각 단계에 적합한 에이전트/스킬을 매핑하는 실행 계획 수립 에이전트.
- **모델**: sonnet
- **도구**: Read, Glob, Grep, WebSearch
- **카테고리**: meta

## 사용 시점

- 여러 파일을 수정해야 하는 복잡한 작업일 때
- 여러 에이전트가 협업해야 하는 작업일 때
- 여러 관심사(프론트엔드, 백엔드, 설정 등)가 섞인 작업일 때
- 작업 순서와 의존 관계를 명확히 해야 할 때
- 리스크가 있는 변경(마이그레이션, 스키마 변경 등)을 계획할 때

## 사용 예시

- "로그인 기능 전체를 만들어줘 -- API, 프론트 폼, 테스트까지"
- "CRA 프로젝트를 Vite로 마이그레이션하고 PWA도 적용해줘"
- "지금 프로젝트에서 개선해야 할 것들 다 정리하고 우선순위 잡아줘"

## 입력/출력

- **입력**: 복잡한 작업 요청 (여러 관심사가 섞인 요구사항)
- **출력**: 실행 계획 -- 전제 조건/가정, 단계별 작업 목록(에이전트/스킬 매핑, 병렬 가능 여부, 예상 결과), 사전 확인 필요 항목, 커밋 분리 계획

## 관련 에이전트

- **agent-creator** (meta) -- 계획 중 필요한 에이전트가 없을 때 신규 생성
- **skill-creator** (meta) -- 계획 중 필요한 스킬이 없을 때 신규 생성
- **frontend-architect** (frontend) -- 프론트엔드 아키텍처 결정이 포함된 계획
- **rust-backend-architect** (backend) -- 백엔드 아키텍처 결정이 포함된 계획
- **product-planner** (domain) -- 기획 단계가 필요한 작업에서 협업
