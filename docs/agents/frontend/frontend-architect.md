# frontend-architect

## 개요

- **역할**: 프론트엔드 아키텍처 설계 전담 에이전트. 프로젝트 구조, 기술 스택, 렌더링 전략, 번들링 설정 등 아키텍처 수준의 판단과 결정을 담당한다. React 18/19, Next.js App Router, TypeScript, 모노레포, 번들러 관련 스킬을 기반으로 근거 있는 아키텍처 제안을 생성한다.
- **모델**: opus
- **도구**: Read, WebSearch, WebFetch
- **카테고리**: frontend

## 사용 시점

- 프론트엔드 프로젝트 구조를 설계하거나 재구성할 때
- 기술 스택 선택 시 근거 있는 비교가 필요할 때
- 렌더링 전략(SSR/SSG/ISR/CSR) 결정이 필요할 때
- 데이터 페칭 아키텍처를 설계할 때
- 번들링/빌드 전략을 선택할 때
- 모노레포/멀티레포 구조를 결정할 때
- TypeScript 설정 구조를 잡을 때
- 성능 아키텍처(캐싱, 코드 스플리팅, Tree Shaking) 결정이 필요할 때

## 사용 예시

- "Next.js 앱에서 데이터 페칭 전략을 어떻게 설계해야 할까?"
- "모노레포에서 UI 패키지와 앱을 어떻게 구성해야 해?"
- "React Compiler 도입 시 기존 최적화 코드를 어떻게 처리하지?"

## 입력/출력

- **입력**: 아키텍처 관련 질문, 앱 유형(SPA/Next.js/라이브러리), 규모(단일 레포/모노레포), 현재 스택 또는 마이그레이션 방향
- **출력**: 판단 근거가 명시된 아키텍처 제안, 구체적 폴더 구조/설정 파일 예시, 트레이드오프 비교표, 단계별 적용 순서

## 관련 에이전트

- **frontend-developer** (frontend) -- 설계된 아키텍처를 실제 코드로 구현하는 에이전트
- **rust-backend-architect** (backend) -- 풀스택 프로젝트에서 백엔드 아키텍처와 조율할 때
- **planner** (meta) -- 아키텍처 변경을 실행 단계로 분해할 때
